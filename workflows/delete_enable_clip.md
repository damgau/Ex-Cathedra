# Workflow: Stage 6 — Commit Reviewed Timeline (delete disabled clips)

Ripple-delete every timeline span that is dead on **every** track from an editor-reviewed
Premiere export, producing the final tightened cut. This is the "commit the disables" step:
it honours whatever enabled/disabled state the editor left in the XML.

---

## Objective

Produce `OUTPUT/07_final.xml` — `OUTPUT/06_reviewed.xml` with all dead-air spans removed and
the rest rippled left. Success = filler/silence dead-air is gone, multicam **camera switches
are preserved**, no `end <= start` clips, and the reported new duration matches the
removed-span math.

## Required Inputs

- `OUTPUT/06_reviewed.xml` — the timeline exported from Premiere after the editor imported
  `05_fillers.xml`, re-enabled false positives, and disabled anything else to cut (bad takes,
  manual trims). (`--input` to override.) This is an FCP7/xmeml export, **not** generated by
  our tools — see Edge Cases.

## Tools Used

- `tools/delete_enable_clip.py` — computes removable spans and ripples them. Reuses
  `apply_cuts(mode="cut")` from `tools/remove_silence.py`.

## Steps

1. In Premiere, finish reviewing the imported `05_fillers.xml`: re-enable any wrongly-disabled
   clip, disable anything else you want removed. A clip is "kept" if enabled on at least one
   track at that point on the timeline.
2. Export the sequence as Final Cut Pro XML (xmeml) to `OUTPUT/06_reviewed.xml`.
3. Run `python tools/delete_enable_clip.py`.
4. Read the summary: dead-air spans removed (count + seconds), disabled clips deleted vs
   retained (camera switches), and old → new duration.
5. Re-import `OUTPUT/07_final.xml` into Premiere and spot-check a few known filler moments
   and at least one camera switch.

## Expected Outputs

- `OUTPUT/07_final.xml` — the final tightened timeline. Dead-air removed; camera-switch spans
  (one video track disabled, another enabled) retained with their disabled clip intact.

## How "removable" is decided (coverage-complement)

A timeline frame is **kept** if any track has an *enabled* clip covering it. Removable spans
are the complement of the union of all enabled-clip coverage within `[0, timeline_end)`. So:

- A span where **all** tracks are disabled (or empty) → removed. This is exactly how fillers
  (Stage 5) and silence get disabled — uniformly across all 10 tracks.
- A span where only **one** video track is disabled (MAIN CAM off so DIV CAM shows) → kept,
  because DIV's enabled clip still covers it. The disabled MAIN clip is retained as-is.

## Edge Cases & Known Issues

- **The input is a Premiere export, not our XML.** Premiere may reorder clipitems, renumber
  ids, or add `<filter>` blocks. The tool keys only off each clipitem's `start` / `end` /
  `in` / `out` / `enabled` (a clip is enabled unless it carries `<enabled>FALSE</enabled>`),
  so element ordering doesn't matter. **Still validate against a real Premiere export** the
  first time — the logic was verified on a copy of `05_fillers.xml` and a synthetic multicam
  fixture, not yet on an actual round-trip.
- **No fragment re-merge.** After Stage 5's mute splits and this stage's ripple, the final
  timeline has many small adjacent clipitems from the same source (≈1740 on the test file,
  same as the old direct-cut path). They are contiguous and play correctly; merging them back
  is a possible future cleanup, not required.
- **`<link>` groups are not maintained** across the clones `apply_cuts` creates — inherited
  from Stage 3, harmless for playback.
- **Generic tool.** `delete_enable_clip.py` only cares about enabled/disabled state, so it can
  commit the disables from any mute-based stage, not just fillers.
- **Frame rate hardcoded to 25 fps** for the duration printout.
